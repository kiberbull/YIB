### Модель нарушителя для АО "Банк Акционерный Кредитный Стабильный"

#### **1. Классификация нарушителей**

1. **Внешние злоумышленники (киберпреступники):**
   - **Мотивы:** Кража финансовых данных клиентов, вымогательство (вредоносное ПО, программы-вымогатели), нарушение доступности сервисов (DDoS).
   - **Возможности:** Использование современных инструментов атак (вредоносное ПО, ботнеты, инструменты для фишинга), анализ и эксплуатация уязвимостей систем.

2. **Инсайдеры (сотрудники и подрядчики):**
   - **Мотивы:** Личное обогащение, месть, корпоративный шпионаж.
   - **Возможности:** Прямой доступ к системам, привилегированный доступ к конфиденциальным данным, возможность саботажа или утечки информации.

3. **Конкуренты:**
   - **Мотивы:** Получение конкурентного преимущества, дискредитация банка.
   - **Возможности:** Использование инсайдеров, кибершпионаж, компрометация подрядчиков.

4. **Хактивисты:**
   - **Мотивы:** Протесты против политики банка, публичная дискредитация.
   - **Возможности:** DDoS-атаки, взлом веб-сайтов банка, раскрытие данных для воздействия на репутацию.

5. **Нарушители с физическим доступом:**
   - **Мотивы:** Кража конфиденциальных данных, выведение из строя оборудования.
   - **Возможности:** Доступ к серверным помещениям, манипуляция физическими устройствами (банкоматами).

---

#### **2. Средства и методы нарушителей**

1. **Технические средства:**
   - Вредоносное ПО (вирусы, трояны, программы-вымогатели).
   - Инструменты для атак на веб-приложения (SQL-инъекции, XSS).
   - DDoS-ботнеты.
   - Средства анализа сетевого трафика.

2. **Методы социального воздействия:**
   - Фишинговые атаки (рассылка поддельных писем для кражи данных).
   - Манипуляция доверенными лицами для получения конфиденциальной информации.

3. **Физические возможности:**
   - Неавторизованный доступ в серверные помещения.
   - Подключение к корпоративной сети через физические порты.
   - Кража устройств, содержащих конфиденциальные данные.

---

#### **3. Потенциальные действия нарушителей**

1. **Компрометация учетных данных:**
   - Кража паролей через фишинг или вредоносное ПО.
   - Использование слабых паролей или отсутствия двухфакторной аутентификации.

2. **Перехват данных в сети:**
   - Атаки типа MITM ("человек посередине").
   - Нарушение целостности данных при передаче.

3. **Физический доступ:**
   - Кража серверного оборудования.
   - Установка шпионского ПО на устройства.

4. **Атака на веб-приложения:**
   - SQL-инъекции, межсайтовый скриптинг (XSS).
   - Нарушение конфиденциальности клиентских данных.

---

#### **4. Меры противодействия**

- **Организационные меры:**
  - Регулярное обучение сотрудников по вопросам информационной безопасности.
  - Политика минимального доступа (least privilege).
  - Проверка подрядчиков и поставщиков.

- **Технические меры:**
  - Внедрение многофакторной аутентификации (MFA).
  - Настройка шифрования сетевого трафика (SSL/TLS).
  - Мониторинг действий пользователей с привилегиями.

- **Физическая безопасность:**
  - Контроль доступа к серверным помещениям.
  - Шифрование данных на мобильных устройствах.
  - Установка систем видеонаблюдения.
